# Intent Ingestion: The Normalization Pipeline
### Step 1 of [Intent → Capital → Transaction]

In the Agentic Economy, the gap between AI and Blockchain is a translation problem. Spacedeck Protocol provides the **Normalization Pipeline** required to convert abstract natural language intent into rigid, deterministic capital deployment. This is the first stage of the **Autonomous Settlement Protocol** — transforming raw agent prose into a structured GoldenPayload ready for authorization and settlement.

## The Problem: Imperative Friction
Current Web3 frameworks require agents to construct raw transactions (imperative). This forces developers to manage high-entropy variables:
*   Real-time gas estimation.
*   Dynamic priority fees and mempool congestion.
*   Slippage calculation and MEV predation.

## The Solution: Declarative Intent
Spacedeck shifts the industry to **Intent Delegation**. The agent declares the desired financial outcome, and the protocol processes that intent through three deterministic state-locks:

1.  **Normalization (NLP -> Schema):** The chaotic input of a natural language prompt is collapsed into a **Standardized Execution Vector** (JSON).
2.  **Validation (Risk Control):** The intent is audited against institutional-grade sanctions and risk parameters via the **Compliance Guardian**.
3.  **Settlement (Competitive Execution):** The rigid schema is handed to a competitive Solver Network, which guarantees the execution price and absorbs the network gas costs.

### Deterministic State Transition
*   **Prose Input:** "I need to liquidate my 50% BONK position into USDC via TWAP over the next 4 hours."
*   **Standardized Vector:** `{"execution_id": "sd_0x1", "asset": "USDC", "vector": "TWAP_LIQUIDITY_SINK", "mass": 50000}`

By utilizing the Normalization Pipeline, your AI agent operates at the level of logic, while Spacedeck handles the physical machine code.
