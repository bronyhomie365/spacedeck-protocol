# Nuclear Shield: Dual-EIP Security Model

Spacedeck Protocol utilizes a two-layer cryptographic handshake to physically decouple **Intent Authorization** from **Capital Approval**. This is the foundation of our "Nuclear Shield" security model for **Autonomous Transaction Settlement**.

## The Institutional Handshake

### 1. Capital Authorization (EIP-2612 Permit)
The Portfolio Manager signs a gasless **ERC-20 Permit** *once*. This mathematical seal authorizes the Spacedeck Clearinghouse to deploy a specific token volume (e.g., $1M USDC) within a defined window. 
*   **Engineering Reality:** This is a pre-signed token approval that resides on-chain or in the clearinghouse vault. No capital leaves the origin vault until a strike is executed.

### 2. Intent Authorization (EIP-712 Handshake)
When the AI Agent identifies a target execution vector, it generates a **GoldenPayload** (Canonical Intent Schema). To execute, the Spacedeck Sentinel requires a second signature: the **Intent Handshake**.
*   **Standard:** Spacedeck uses **EIP-712** (Typed Structured Data) to sign the specific intent (e.g., "Liquidate 50k USDC for SOL").
*   **Risk Control:** The AI Agent manages the *Intent*, but the *Capital* remains locked behind the EIP-2612 wall.

## Security Advantages
*   **Zero Local Key Storage:** AI Agents hold scoped Spacedeck API keys, never the actual capital keys.
*   **Deterministic Guardrails:** Institutional users set the "Maximum Exposure" (The Permit), while Agents manage the "Frequency" (The Intent).
*   **Gasless Revocation:** Budgets and Intent permissions can be revoked without Layer 1 friction.

The Dual-EIP model ensures that Spacedeck is the safest clearinghouse for institutional-scale **Autonomous Transaction Settlement**.
